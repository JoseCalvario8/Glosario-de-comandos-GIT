Autor: José Calvario Ramírez

Comandos de Git
Comando Descripción Parámetros Ejemplo
git init Inicializa un nuevo repositorio Git en un directorio. Ninguno. git init
git clone Clona un repositorio Git existente en un nuevo directorio. URL del repositorio remoto. git clone https://github.com/ejemplo/repo.git
git add Agrega cambios al área de preparación (staging). Nombre de archivos o directorios. git add archivo.txt
git commit Registra los cambios en el repositorio. -m (mensaje del commit). git commit -m "Añadir función de inicio de sesión"
git branch Lista las ramas en el repositorio. -a (muestra todas las ramas). git branch -a
git checkout Cambia de rama o restaura archivos. Nombre de la rama o archivo. git checkout nueva-funcionalidad
git merge Fusiona una rama con la rama actual. Nombre de la rama a fusionar. git merge feature-branch
git status Muestra el estado actual del repositorio. Ninguno. git status
git log Muestra el historial de commits. Varios para personalizar la salida. git log --oneline
git remote Muestra los repositorios remotos configurados. Ninguno. git remote -v
git fetch Descarga objetos y referencias de otro repositorio. Ninguno. git fetch origin
git remote add Agrega un nuevo repositorio remoto. Nombre del repositorio remoto y URL. git remote add upstream https://github.com/upstream/repo.git
git diff Muestra las diferencias entre el área de preparación y el directorio de trabajo. Ninguno o nombre de archivo. git diff
git reset Reinicia el HEAD a un commit específico, deshaciendo commits anteriores. Nombre del commit o --hard, --soft, --mixed. git reset HEAD~1
git remote remove Elimina un repositorio remoto. Nombre del repositorio remoto. git remote remove upstream
git stash Guarda temporalmente los cambios no comprometidos en una pila de cambios. Ninguno. git stash
git tag Etiqueta commits para marcar versiones específicas. Nombre de la etiqueta y commit. git tag v1.0
git rebase Reorganiza commits aplicándolos sobre otra rama. Nombre de la rama objetivo. git rebase main
git cherry-pick Aplica un commit específico de una rama a otra. SHA del commit. git cherry-pick abc123
git log --graph Muestra el historial de commits en un formato gráfico. Ninguno. git log --graph
git blame Muestra quién modificó cada línea de un archivo. Nombre del archivo. git blame archivo.txt
git clean Elimina archivos no rastreados en el directorio de trabajo. Parámetros como -n (simulación) y -f (forzar). git clean -n
git config Configura opciones de Git a nivel global o local. Parámetros como --global o --local. git config --global user.name "Tu Nombre"
git remote show Muestra información detallada sobre un repositorio remoto. Nombre del repositorio remoto. git remote show origin
git revert Crea un nuevo commit que revierte los cambios de un commit anterior. SHA del commit. git revert abc123
git bisect Ayuda a encontrar un commit que introdujo un problema, realizando una búsqueda binaria. Parámetros como start y good/bad. git bisect start

Conclusion personal:

Git es un potente controlador de versiones que como ya vimos en la tabla, cuenta con una gran cantidad de comandos que nos sirven para hacer/controlar todas nuestras necesidades en nuestros archivos, esto con el fin de que nos ayude a llevar un mejor control de lo que estamos haciendo.
